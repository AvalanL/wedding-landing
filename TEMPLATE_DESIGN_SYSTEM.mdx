## System Architecture

### Overview
The wedding template system is built on **Next.js 14** with **Tailwind CSS** and **shadcn/ui** components. Each template is a self-contained React component that receives standardized props and renders a complete wedding website.

### File Structure
```
app/
  page.tsx              # Main file containing all templates and editor
  layout.tsx            # Global layout with fonts
  globals.css           # Tailwind configuration
components/ui/          # shadcn components (Button, Input, etc.)
public/                 # Static images
```

```typescript
interface WeddingData {
```

## Design Principles

### 1. Emotional Resonance
Each template evokes specific feelings appropriate to its theme:
- **Romantic**: Love, warmth, softness
- **Modern**: Sophistication, clean lines, contemporary
- **Vintage**: Nostalgia, timelessness, heritage
- **Elegant**: Luxury, refinement, minimalism
- **Bohemian**: Nature, freedom, earthiness
- **Minimalist**: Simplicity, space, clarity
- **Garden**: Growth, freshness, botanical beauty
- **Art Deco**: Glamour, geometry, 1920s opulence
- **Beach**: Ocean, relaxation, destination
- **Rustic**: Countryside, warmth, authenticity
- **Whimsical**: Fun, playfulness, personality
- **Cinematic**: Drama, photography, editorial

### 2. Visual Hierarchy
- **Hero section**: Largest text (text-4xl md:text-5xl), full-screen height
- **Section headings**: Large text (text-4xl), centered or left-aligned
- **Body text**: Readable sizes (text-lg to text-xl), generous line-height
- **Details**: Smaller text (text-sm to text-base), uppercase tracking

### 3. Consistency Within Templates
- Repeating design patterns create trust
- Consistent spacing system (multiples of 4px)
- Unified color palette throughout
- Cohesive typography choices

### 4. Accessibility First
- WCAG AAA contrast ratios where possible
- Minimum 44x44px tap targets
- Semantic HTML structure
- Focus states on interactive elements

### 5. Performance Minded
- CSS transforms for animations (GPU accelerated)
- Opacity transitions (GPU accelerated)
- No layout thrashing
- Minimal JavaScript

## Color Palette System

### Philosophy
Each template has **3 carefully curated color palettes** that work harmoniously with its design language. Palettes use **Tailwind CSS color classes** for consistency and reliability.

### Color Role Definitions

#### 1. Primary Color
**Purpose**: Main brand color, used for:
- Key interactive elements (buttons, links)
- Icons and decorative elements
- Important text highlights
- Timeline markers, borders

**Usage Pattern**:
```jsx
<Button className={`bg-${palette.primary} hover:bg-${palette.primary}/90`}>
<Calendar className={`text-${palette.primary}`} />
<div className={`border-${palette.primary}`}></div>
```

#### 2. Secondary Color
**Purpose**: Supporting color, used for:
- Secondary buttons
- Gradient combinations with primary
- Alternative highlights
- Decorative elements

**Usage Pattern**:
```jsx
<div className={`bg-gradient-to-br from-${palette.primary} to-${palette.secondary}`}>
<span className={`text-${palette.secondary}`}>Supporting text</span>
```

#### 3. Accent Color
**Purpose**: Highlight color, used for:
- Hover states
- Call-to-action elements
- Decorative icons
- Tertiary elements

**Usage Pattern**:
```jsx
<Heart className={`text-${palette.accent} animate-pulse`} />
<div className={`border-${palette.accent}/40`}></div>
```

#### 4. Background Color
**Purpose**: Page and section backgrounds, used for:
- Main page background
- Card backgrounds
- Section alternating backgrounds
- Subtle color washes

**Usage Pattern**:
```jsx
<div className={`bg-${palette.background}`}>
<section className={`bg-gradient-to-b from-white to-${palette.background}`}>
```

#### 5. Text Color
**Purpose**: Main text color, used for:
- All body text
- Headings
- Paragraphs
- Must have proper contrast with background

**Usage Pattern**:
```jsx
<h1 className={`text-${palette.text}`}>Heading</h1>
<p className={`text-${palette.text}/90`}>Body with 90% opacity</p>
```

### Color Palette Rules

#### Rule 1: Light Backgrounds Need Dark Text
```javascript
// ‚úÖ CORRECT
background: "rose-50",    // Light background
text: "rose-900",         // Dark text (high contrast)

// ‚ùå INCORRECT
background: "rose-50",    // Light background
text: "rose-100",         // Light text (no contrast)
```

#### Rule 2: Dark Backgrounds Need Light Text
```javascript
// ‚úÖ CORRECT
background: "gray-900",   // Dark background
text: "gray-50",          // Light text (high contrast)

// ‚ùå INCORRECT
background: "gray-900",   // Dark background
text: "gray-900",         // Dark text (no contrast)
```

#### Rule 3: Use Valid Tailwind Colors Only
```javascript
// ‚úÖ CORRECT
"amber-600", "rose-500", "teal-400", "slate-900"

// ‚ùå INCORRECT
"mint-400",    // Tailwind has no "mint" color
"stone-600",   // Use "stone" carefully, prefer standard colors
```

#### Rule 4: Primary Should Be 500-700 Range
```javascript
// ‚úÖ CORRECT - Visible and vibrant
primary: "rose-600"

// ‚ö†Ô∏è CAUTION - Too light or too dark
primary: "rose-200"  // May not have enough contrast
primary: "rose-950"  // May be too dark
```

#### Rule 5: Gradients Use Primary + Secondary
```javascript
// ‚úÖ CORRECT PATTERN
<div className={`bg-gradient-to-br from-${palette.primary} to-${palette.secondary}`}>

// Common gradient patterns:
from-${palette.background} via-white to-${palette.background}
from-${palette.primary}/20 via-${palette.secondary}/30 to-${palette.accent}/40
```

## Creating New Templates

### Step-by-Step Template Creation Guide

#### Step 1: Define Template Concept

**Questions to answer:**
1. What wedding style does this serve? (Cultural, venue-type, aesthetic)
2. What emotions should it evoke?
3. What's the target couple demographic?
4. What makes it unique from existing templates?

**Example:**
```
Template: "Desert/Southwest"
Style: Southwestern, desert wedding, bohemian-luxe
Emotions: Warmth, adventure, natural beauty
Demographics: Couples getting married in Arizona, New Mexico, desert venues
Unique: Terracotta colors, cacti motifs, sunset gradients
```

#### Step 2: Design Color Palettes

Create **exactly 3 palettes** for the template:

```javascript
templatename: [
  {
    name: "Palette Name in Swedish",  // e.g., "√ñken Solnedg√•ng"
    primary: "color-600",              // Main interactive color (500-700 range)
    secondary: "color-500",            // Supporting color
    accent: "color-400",               // Highlight color (lighter)
    background: "color-50",            // Light background (50-100)
    text: "color-900",                 // Dark text (800-950) for light bg
  },
  // Two more palettes...
]
```

**Palette Rules:**
- Use only valid Tailwind colors
- Ensure high contrast (light bg ‚Üí dark text, dark bg ‚Üí light text)
- Primary should be most saturated (500-700)
- Background should be subtle (50-100 or 800-950)
- Test all 5 colors work together visually

**For Dark Backgrounds:**
```javascript
{
  background: "slate-900",            // Dark (900)
  text: "slate-50",                   // Light text (50-100)
  primary: "blue-400",                // Lighter colors show on dark
  secondary: "indigo-400",
  accent: "cyan-300",
}
```

#### Step 3: Choose Typography

**Select font family:**
```javascript
// Serif (traditional, elegant, romantic)
fontFamily: 'Georgia, serif'
fontFamily: 'Playfair Display, serif'

// Sans-serif (modern, clean, contemporary)
fontFamily: 'Helvetica Neue, Arial, sans-serif'
fontFamily: 'Source Sans 3, sans-serif'

// Display (unique, playful, distinctive)
fontFamily: 'Comic Sans MS, cursive, sans-serif'
```

**Define font weights:**
```javascript
font-thin       // 100 - Ultra-minimalist, elegant
font-light      // 300 - Modern, airy
font-normal     // 400 - Standard body
font-semibold   // 600 - Emphasis
font-bold       // 700 - Strong, playful
```

**Set letter-spacing:**
```javascript
tracking-tight      // -0.025em - Tight headings
tracking-normal     // 0em - Default
tracking-wide       // 0.025em - Elegant
tracking-widest     // 0.1em - Uppercase labels
tracking-[0.2em]    // Custom extreme (cinematic)
```

## Best Practices

### 1. Always Use Palette Variables

```jsx
// ‚úÖ CORRECT
<div className={`text-${palette.primary} bg-${palette.background}`}>

// ‚ùå INCORRECT
<div className="text-rose-600 bg-rose-50">
```

**Exception**: Whimsical template uses hardcoded multi-colors intentionally for playful effect.

### 2. Check Section Visibility

```jsx
// ‚úÖ CORRECT
{sections.hero && (
  <section>...</section>
)}

// ‚ùå INCORRECT
<section>...</section>  // Always renders
```

### 3. Provide Image Fallbacks

```jsx
// ‚úÖ CORRECT
uploadedImages["hero"] || data.gallery[0]?.url || "/default.jpg"

// ‚ùå INCORRECT
uploadedImages["hero"]  // May be undefined
```

### 4. Use Semantic HTML

```jsx
// ‚úÖ CORRECT
<section>
  <h2>Section Title</h2>
  <article>...</article>
</section>

// ‚ùå INCORRECT
<div>
  <div>Section Title</div>
  <div>...</div>
</div>
```

### 5. Implement Hover States

```jsx
// ‚úÖ CORRECT
<div className="hover:scale-110 hover:shadow-xl transition-all duration-300">

// ‚ùå INCORRECT
<div className="scale-110">  // No interaction feedback
```

### 6. Use Group Hover Patterns

```jsx
// ‚úÖ CORRECT
<div className="group">
  <Icon className="group-hover:scale-110" />
  <Text className="group-hover:text-primary" />
</div>

// Less optimal
<div className="hover:scale-110">
  <Icon className="hover:scale-110" />  // Both need hover state
  <Text className="hover:text-primary" />
</div>
```

### 7. GPU-Accelerated Animations

```jsx
// ‚úÖ CORRECT - GPU accelerated
transform
opacity
translate
scale
rotate

// ‚ùå AVOID - Layout thrashing
margin
padding
width
height
top/left/bottom/right
```

### 8. Consistent Spacing Rhythm

```jsx
// ‚úÖ CORRECT - Follows 4px system
py-16  // 64px
mb-10  // 40px
gap-8  // 32px
p-6    // 24px

// ‚ùå INCORRECT - Breaks rhythm
py-17  // 68px (not in system)
mb-11  // 44px (not in system)
```

### 9. Responsive Design Patterns

```jsx
// ‚úÖ CORRECT
<h1 className="text-4xl md:text-5xl">
<div className="grid grid-cols-1 md:grid-cols-3 gap-8">

// ‚ùå INCORRECT
<h1 className="text-5xl">  // Too large on mobile
<div className="grid grid-cols-3">  // Breaks on mobile
```

### 10. Accessibility

```jsx
// ‚úÖ CORRECT
<img src={photo.url} alt={photo.alt} />
<button className="min-w-[44px] min-h-[44px]">  // Tap target
<a className="focus:ring-2 focus:ring-offset-2">

// ‚ùå INCORRECT
<img src={photo.url} />  // No alt text
<button className="w-8 h-8">  // Too small
<a>  // No focus state
```

### 11. Performance

```jsx
// ‚úÖ CORRECT
<img loading="lazy" />
<div className="transform-gpu">  // Forces GPU

// Future optimization
<Image src={photo.url} width={800} height={600} />  // Next.js Image
```

### 12. Testing Checklist

Before considering a template complete:

- [ ] All 3 palettes render correctly
- [ ] Light palettes have dark text (good contrast)
- [ ] Dark palettes have light text (good contrast)
- [ ] All sections respect `sections` visibility prop
- [ ] Images have proper fallbacks
- [ ] Hover states work on all interactive elements
- [ ] Mobile responsive (test at 375px, 768px, 1024px)
- [ ] Animations are smooth (60fps)
- [ ] No console errors
- [ ] Text is readable on all backgrounds
- [ ] Icons scale properly on hover
- [ ] Cards have proper shadows/depth
- [ ] Spacing feels consistent
- [ ] Typography hierarchy is clear
- [ ] Template has unique personality
- [ ] Matches design philosophy
